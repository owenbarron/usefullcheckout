How It Works

The Enforcement Timeline

1
Container Marked Missing
A nightly cron job finds containers in "Returned" status with no new checkout in X days (default 30). They're automatically flagged as Missing in the database.
2
Repeat Offender Warning
Users with Y+ missing containers (default 3) are flagged as Repeat Offenders and sent a warning email listing their missing containers and the deadline to return them.
3
Account Frozen
After Z days (default 14) with no action, the cron job automatically freezes the user's account. A "You've Been Frozen" email is sent with instructions to resolve.
4
Charge & Unlock
Admin charges the user via Stripe or Campus Card. Containers are marked as "Purchased" and the account is unlocked. If the user returns containers below the threshold, auto-unlock triggers.
All thresholds (X, Y, Z) are configurable per campus via the admin panel.

Phase 1 — Full Breakdown

CCC-2821 (Epic)
Identification & Automation
  • Add returnStatus field on assignments (Pending → Complete → Missing) to track container lifecycle without changing existing "returned" logic
  • Nightly cron job scans all sites for containers in "Returned" status with no new checkout in X days (default 30) and marks them as Missing
  • Database fields to track: when a user was warned, whether account is locked, and that the lock is specifically for missing containers
Account Management
  • Nightly cron job identifies Repeat Offenders (users with Y+ missing containers, default 3) and records the date they were warned
  • Automated warning email sent to repeat offenders with full list of missing containers and deadline to return them
  • After Z days (default 14) with no action, cron job automatically freezes the user's account
  • "You've Been Frozen" email triggered with instructions on how to get unfrozen
Unlocking & Fees
  • Nightly cron job checks locked users and auto-unlocks if their missing container count drops below the threshold
  • Customer service can manually charge users via Stripe invoice for missing containers
  • Support can manually charge via Campus Card API for schools using campus payment systems
  • Manual unlock after charge — set containers + assignments to "Purchased" status so they're no longer flagged
  • System will not re-lock a manually unlocked user unless they accumulate new missing containers

Phase 2 — Full Breakdown

User Experience
  • Missing container assignments show up in Active Rentals so users can see what they owe
  • Login popup warning for users with 1+ missing containers: "A container you scanned to return has been identified as missing..."
  • Push notifications for repeat offenders about missing containers
  • In-app warning that account is at risk of being frozen
  • User can trigger an in-app recalculation of missing containers to automatically unlock their account after returning items
  • Email notification when user moves from Locked to Unlocked status
Admin Panel & Settings
  • Admin settings UI to toggle "Track Missing Containers" and "Lock Users for Missing Containers" independently
  • Configurable thresholds per campus: Days until missing (X), Lock threshold (Y containers), Grace period (Z days)
  • Exception periods for semester breaks — pause enforcement during summer and winter break with named date ranges
  • During exception periods: no new containers marked missing, no new warnings or freezes. Existing freezes remain. Return detection still runs.
Advanced Charging & Reporting
  • Admin charging modal: select specific containers to charge for, with per-container-type pricing (e.g. cups $17, bowls $25)
  • Flexible discount field — admin can arbitrarily reduce the total if user makes a plausible claim about some containers
  • Separate line items per container plus discount line, all recorded in the fee ledger
  • Reports showing count of missing containers and repeat offenders per school
  • Reminder emails to repeat offenders before freeze deadline
